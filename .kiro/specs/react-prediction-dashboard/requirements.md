# Requirements Document

## Introduction

This feature adds a React.js frontend (built with Vite) and a FastAPI backend to the existing World Cup Predictor project. The dashboard displays World Cup 2026 group stage standings and knockout bracket structure alongside real-time match predictions generated by the existing Python prediction engine. The FastAPI backend serves as an intermediary layer that fetches live data from football-data.org, runs the prediction pipeline, and exposes structured REST endpoints for the React frontend to consume.

## Glossary

- **Dashboard**: The React.js single-page application that renders group stage and knockout bracket views
- **Backend**: The FastAPI Python server that exposes REST endpoints for the Dashboard
- **Prediction_Engine**: The existing `src/predictor.py` module that trains a model and produces match outcome probabilities
- **Football_Data_API**: The football-data.org v4 REST API that provides team, match, and competition data
- **Group_Stage_View**: The Dashboard tab displaying all World Cup groups (A through L) with teams and predictions
- **Knockout_Bracket_View**: The Dashboard tab displaying the tournament bracket from Round of 32 through the Final
- **Team_Card**: A visual component displaying a team's crest image and name
- **Match_Prediction_Card**: A visual component displaying two teams and their respective win probabilities
- **Crest_URL**: The SVG image URL for a team's flag/crest provided by the Football_Data_API

## Requirements

### Requirement 1: FastAPI Backend Server

**User Story:** As a developer, I want a FastAPI backend server that exposes REST endpoints, so that the React frontend can fetch predictions and tournament data.

#### Acceptance Criteria

1. THE Backend SHALL expose a REST API on a configurable port (default 8000)
2. WHEN the Dashboard sends a GET request to `/api/groups`, THE Backend SHALL return JSON containing all World Cup 2026 groups with team names, team IDs, and Crest_URLs for each team in the group
3. WHEN the Dashboard sends a GET request to `/api/predictions`, THE Backend SHALL return JSON containing match predictions with home_team_name, away_team_name, home_win_prob, home_loss_prob, home_team_crest, away_team_crest, and match_date for each scheduled match
4. WHEN the Dashboard sends a GET request to `/api/knockout`, THE Backend SHALL return JSON containing the knockout bracket structure with round names, match slots, and team information (or TBD placeholders for undetermined teams)
5. THE Backend SHALL read the API key from the FOOTBALL_DATA_API_KEY environment variable
6. IF the Football_Data_API returns an error, THEN THE Backend SHALL return an appropriate HTTP error response with a descriptive message to the Dashboard
7. THE Backend SHALL enable CORS to allow requests from the frontend development server origin

### Requirement 2: Group Stage Data Endpoint

**User Story:** As a frontend developer, I want a groups endpoint that returns structured group data, so that the Dashboard can render all 12 World Cup groups.

#### Acceptance Criteria

1. WHEN the Backend receives a request to `/api/groups`, THE Backend SHALL fetch standings data from the Football_Data_API for competition ID 2000
2. THE Backend SHALL return groups labeled A through L, each containing exactly 4 teams
3. FOR each team in a group, THE Backend SHALL include the team_id, team_name, and crest field containing the SVG URL from the Football_Data_API
4. IF the Football_Data_API returns incomplete group data, THEN THE Backend SHALL return available groups and indicate missing data in the response

### Requirement 3: Predictions Endpoint

**User Story:** As a user, I want to see match predictions with win probabilities, so that I can assess likely outcomes for upcoming World Cup matches.

#### Acceptance Criteria

1. WHEN the Backend receives a request to `/api/predictions`, THE Backend SHALL invoke the Prediction_Engine with competition ID 2000
2. THE Backend SHALL return predictions sorted by match_date in ascending order
3. FOR each prediction, THE Backend SHALL include home_team_name, away_team_name, home_win_prob, home_loss_prob, home_team_crest, away_team_crest, and match_date
4. THE Backend SHALL express home_win_prob and home_loss_prob as floating-point numbers between 0.0 and 1.0
5. IF no scheduled matches exist, THEN THE Backend SHALL return an empty predictions array with HTTP status 200

### Requirement 4: Knockout Bracket Endpoint

**User Story:** As a user, I want to see the knockout tournament bracket, so that I can understand the path to the final for each team.

#### Acceptance Criteria

1. WHEN the Backend receives a request to `/api/knockout`, THE Backend SHALL fetch match data from the Football_Data_API for competition ID 2000 and filter for knockout stage matches
2. THE Backend SHALL organize knockout matches into rounds: Round_of_32, Round_of_16, Quarter_Finals, Semi_Finals, and Final
3. FOR each knockout match with determined teams, THE Backend SHALL include team_name and crest for both home and away teams
4. FOR each knockout match with undetermined teams, THE Backend SHALL include a placeholder value of "TBD" for team_name and an empty string for crest
5. THE Backend SHALL include the match status (SCHEDULED, TIMED, IN_PLAY, FINISHED) for each knockout match
6. WHEN a knockout match has status FINISHED, THE Backend SHALL include the final score

### Requirement 5: React Frontend Application

**User Story:** As a user, I want a web-based dashboard to view World Cup predictions, so that I can access predictions from a browser.

#### Acceptance Criteria

1. THE Dashboard SHALL be a React.js application initialized with Vite in a `frontend/` subdirectory of the project
2. THE Dashboard SHALL provide tab-based navigation between the Group_Stage_View and the Knockout_Bracket_View
3. THE Dashboard SHALL fetch data from the Backend on initial load and display a loading indicator while data is being retrieved
4. IF the Backend returns an error, THEN THE Dashboard SHALL display an error message to the user with a retry option
5. THE Dashboard SHALL be responsive and render correctly on viewport widths from 320px to 1920px

### Requirement 6: Group Stage View

**User Story:** As a user, I want to see all World Cup groups with their teams and match predictions, so that I can understand the group stage landscape.

#### Acceptance Criteria

1. THE Group_Stage_View SHALL display all groups (A through L) in a grid or list layout
2. FOR each group, THE Group_Stage_View SHALL display the group name as a header (e.g., "Group A")
3. FOR each team in a group, THE Group_Stage_View SHALL display a Team_Card containing the team crest image and team name
4. THE Group_Stage_View SHALL display Match_Prediction_Cards for matches between teams within the same group
5. FOR each Match_Prediction_Card, THE Group_Stage_View SHALL display the home team name, away team name, and the win probability percentage for each team
6. THE Group_Stage_View SHALL render team crest images from the Crest_URL as inline SVG or img elements with appropriate alt text for accessibility

### Requirement 7: Knockout Bracket View

**User Story:** As a user, I want to see the tournament knockout bracket, so that I can follow the elimination path from Round of 32 to the Final.

#### Acceptance Criteria

1. THE Knockout_Bracket_View SHALL display the bracket structure organized by round (Round_of_32, Round_of_16, Quarter_Finals, Semi_Finals, Final)
2. FOR each match slot in the bracket, THE Knockout_Bracket_View SHALL display the team names and crests for determined teams
3. FOR each match slot with undetermined teams, THE Knockout_Bracket_View SHALL display "TBD" as the team name
4. WHEN a knockout match has status FINISHED, THE Knockout_Bracket_View SHALL display the final score alongside the team names
5. THE Knockout_Bracket_View SHALL visually connect match slots between rounds to indicate progression paths

### Requirement 8: Team Display Component

**User Story:** As a user, I want to see each team represented by their flag and name, so that I can quickly identify teams visually.

#### Acceptance Criteria

1. THE Team_Card SHALL display the team crest image loaded from the Crest_URL
2. THE Team_Card SHALL display the team name adjacent to the crest image
3. IF the Crest_URL is empty or the image fails to load, THEN THE Team_Card SHALL display a fallback placeholder icon
4. THE Team_Card SHALL include an alt attribute on the crest image containing the team name for screen reader accessibility

### Requirement 9: Prediction Probability Display

**User Story:** As a user, I want to see win probabilities as percentages, so that I can understand the likelihood of each outcome at a glance.

#### Acceptance Criteria

1. THE Match_Prediction_Card SHALL display win probabilities as percentages rounded to one decimal place (e.g., "65.3%")
2. THE Match_Prediction_Card SHALL display probabilities for both the home team and the away team
3. THE Match_Prediction_Card SHALL visually distinguish the favored team (the team with higher win probability)
4. THE Match_Prediction_Card SHALL display the match date in a human-readable format

### Requirement 10: Real-Time Data Refresh

**User Story:** As a user, I want the dashboard to reflect current tournament data, so that predictions stay up to date as matches are played.

#### Acceptance Criteria

1. THE Dashboard SHALL fetch fresh data from the Backend each time the user navigates to a tab
2. WHEN the user clicks a refresh button, THE Dashboard SHALL re-fetch data from the Backend and update the displayed content
3. WHILE data is being refreshed, THE Dashboard SHALL display a loading indicator without clearing the previously displayed content
